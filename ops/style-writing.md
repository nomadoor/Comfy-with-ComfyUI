# Writing Style Guide
 
 このドキュメントは、プロジェクトにおける文章作成のルールをまとめたものです。
 
 ## 基本方針
 
 - **自然な日本語**: 直訳調を避け、読みやすい日本語を心がける。
 - **過度な強調を避ける**: 太字の多用を控え、必要な箇所のみ強調する。
 - **専門用語**: 無理に日本語化せず、英語のまま使用する（例: Workflow, Checkpoint）。ただし、ユーザーが混乱しないよう配慮する。
 
### 文体
- **敬体（です・ます調）**: 基本的に「〜です」「〜ます」を使用する。ただし、手順の箇条書きなどでは**体言止め**（例：「〜をクリック」「〜を選択」）を用いて簡潔にする。
- **親しみやすさ**: 堅苦しい技術文書ではなく、読者に語りかけるようなトーンを意識する。
- **簡潔さ**: 「操作：」のような冗長なラベルは省略する。
- **ユーザー視点**: 読者がつまずきそうなポイントを先回りしてフォローするなど、寄り添う姿勢で書く。 ## フォーマット
 
### 見出し
- **階層**: `####` 以下の深い階層は避ける。箇条書きやテーブルを活用して構造をフラットに保つ。

### リスト（箇条書き）

- リスト記号にはハイフン `-` を使用する（`*` は使用しない）。
- **番号付きリスト**: `<ol>` タグ（`1.` など）は使用しない。順序が必要な場合も `- 1.` `- 2.` のように手書き番号で表現する。
- インデントは **スペース2つ** で統一する。
- ネスト（入れ子）する場合も、親要素に対してスペース2つ分インデントする。

```markdown
- 親アイテム
  - 子アイテム
    - 孫アイテム
```

### コードブロック

- リスト内にコードブロックを含める場合は、インデントを揃える。

```markdown
- 手順の説明
  ```text
  コードやパス
  ```
```

## 用語統一

- **workflow**: 原則として小文字の `workflow` を使用する（文頭など、文法的に必要な場合を除くが、基本は小文字推奨）。
- **ComfyUI**: 表記は `ComfyUI` で統一。

## 画像・動画 (Gyazo)

Gyazoを使用し、適切な属性を付与する。

### 画像

通常の画像には `{gyazo=image}` を付与する。

```markdown
![](https://i.gyazo.com/hash.png){gyazo=image}
```

### 動画（ループ）

短い動画やループ再生させたいGIF/MP4には `{gyazo=loop}` を付与する。
プレイヤーとして再生させたい動画には`{gyazo=player}`を付与する。

```markdown
![](https://gyazo.com/hash){gyazo=loop}
![](https://gyazo.com/hash){gyazo=player}
```

## 著者性

著者は人間オーナーであり、エージェントは軽い校正を超える本文の改変を行わない。
