# Writing Style Guide

このドキュメントは、プロジェクトにおける文章作成ルールをまとめたものです。
実際の ComfyUI 解説記事の文体・構造・語り口を忠実に再現し、読みやすく統一されたドキュメントを維持することを目的とします。

## 01. 基本方針

- **自然な日本語**: 翻訳調を避け、やわらかく、日常的に読める日本語を書く。
- **必要十分・シンプル**: 解説は最短で役割が分かるように始め、不要に長文化しない。
- **英語用語は原則そのまま**: `Workflow` / `Checkpoint` / `string` / `widget` など、ComfyUI やプログラミングで一般的な語は英語を優先する。必要に応じてカタカナ補足を添えてよい（例: `string`（ストリング））。
- **やわらかい技術文書**: アカデミックではなく、読者に語りかけるような文体を維持する。
  - 例: 「〜なんですね」「〜といった具合です」「〜することができます」
- **著者の視点・軽い評価を許容**: 「これはあまり使われません」「個人的にはこちらを推奨」など、適度に判断を述べてよい。

## 02. 文体

- **敬体（です・ます）** を基本とする。
- 手順や UI 操作は **体言止め** を使い、短くまとめる。
  - 例: 「〜をクリック」「〜を選択」
- 読者がつまづきやすい箇所では、さりげなくフォローする。
  - 例: 「どれを選んでもOKです」「地味ですが重要です」
- 冗長なラベル（例: 「操作：」「注意：」）は使わず、自然な流れで説明する。

## 03. ページ構成ルール

実際のあなたの記事の構造を基準とする。

### 3-1. 見出し構成

- **H2 (`##`)**：大カテゴリ（例: マスクの作成 / マスクの編集 / ノードの追加）
- **H3 (`###`)**：ノード名・操作名、その他
  - 例: `### GrowMask ノード`
- 各項目は「1 ノード 1 見出し」を基本とする。
- **H4 以下は使用しない**。必要なら箇条書きで代替する。

### 3-2. セクション境界

- テーマが大きく変わる箇所では `---` を入れて区切る（あなたの既存記事の慣例に合わせる）。

## 04. 各項目（ノード解説）のテンプレート

あなたの文章パターンを標準化すると以下になる。

```markdown
### ノード名

![](URL){gyazo=image}

[](/workflows/...json)

一文で役割を説明する。

必要なら、パラメータや使いどころを箇条書きで補足する。
- ...
- ...

場合によっては短い Tip・注意書きを追加してもよい。
```

**特徴として重要なルール：**

- **最初の一文で役割を言い切る**（あなたの文体の最大の特色）
- **画像 → workflow → 説明** という「固定の並び」を守る
- 説明文は冗長にせず、1〜3文で完結させる
- 実務的な使いどころは柔らかく補足する
  - 例: 「全面を塗りつぶしたマスクが必要な場合に使用します」

## 05. リスト（箇条書き）

- 記号は必ず **ハイフン `-`** を使う。
- 番号が必要な場合は `- 1.` `- 2.` のように「手書き番号」。
- 入れ子は **スペース 2 つ** でインデント。
- 説明の粒度は短く、1文を長くしない。

例：

```markdown
- 1. ノードを選択
- 2. `Node Selection Toolbox` の `🕸️` をクリック
```

## 06. コードブロック

- リスト内に含める場合はインデントを揃える。
- コード以外（文章）の囲いにコードブロックを使わない。

## 07. 用語の統一

- **workflow**：小文字で統一。
- **ComfyUI**：正式表記 ComfyUI。
- 型・パラメータ・ノード名は backtick で囲む。
  - 例: `MASK` / `IMAGE` / `delimiter` / `String (Multiline)`。

## 08. 画像・動画（Gyazo）

- ノードの静止画 → `{gyazo=image}`
- UI 操作・アニメーション → `{gyazo=loop}`
- プレイヤー再生 → `{gyazo=player}`

テンプレ：

```markdown
![](https://i.gyazo.com/hash.png){gyazo=image}
```

## 09. Tip・補足・注意書き

- 必要な場合は `blockquote` (`>`) を使用。
- トーンはやわらかく断定しすぎない。

例：

> 実際のところ、ミュートはあまり使われません。

## 10. 著者性と編集方針

- 著者は人間オーナーであり、エージェント（AI）は大幅な改変を行わず編集・改善に徹する。
- エージェントが行うのは以下：
  - 誤情報の修正
  - 読みやすい構造への調整
  - 不足している説明の補完提案
  - 用語統一・スタイル統一
